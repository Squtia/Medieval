# 🛡️ Fantasy Idle RPG 系統導覽總覽

這份文件為您整理了截至目前為止，我們所開發與建立的所有核心遊戲系統。這不僅可以幫助您回顧專案進度，也是未來繼續開發的重要參考。

---

## 🗺️ 1. 世界地圖與 UI 架構系統
遊戲採用了**「三層式沉浸導覽架構」**，讓玩家體驗從宏觀世界縮小至微觀建築的過程：
- **世界地圖 (World Map)**：透過節點 (Nodes) 所組成的地圖，包含三大國家與危險的野外區域。
- **國家街道 (Street View)**：當玩家進入所屬國家時，會看到資訊總覽，包含右側的「冒險者名單」、「遊戲日誌（具備分類過濾功能）」，以及左側通往各大設施的「入口大門」。
- **獨立建築內部 (Facility Views)**：四大獨立全螢幕建築，包含背景圖預留位與返回按鈕。

## 🏛️ 2. 領地內政與外交系統
在**「👑 謁見廳 (Hall)」**與背景執行迴圈中負責處理：
- **放置收益引擎**：採用「時間戳記 (Timestamp)」比對而非逐秒倒數，完美支援離線放置結算。只要玩家上線，系統會一瞬間結算錯過的所有稅收與任務。
- **爵位晉升 (Noble Titles)**：平民 ➔ 騎士 ➔ 男爵 ➔ 子爵...等。只要達到特定的「聲望」與「皇家好感度」，系統會自動結算並發布晉升公告。
- **外交互動**：可花費金幣進行「進貢 (+好感度)」或「舉辦宴會 (+聲望)」。

---

## 🦸 3. 冒險者育成系統
複雜且深度的 RPG 屬性模型：
- **隨機生成系統**：結合世界觀的「姓名生成器 (NameGenerator)」，具備特殊的傭兵家族姓氏。
- **六維屬性模型**：力量、敏捷、體質、智慧、精神、幸運。
- **職業與性格組合**：
  - **職業 (Job)** 決定升級時六維的成長係數。
  - **性格 (Trait)** 提供屬性的總百分比乘數，以及經驗值獲取難易度的修正。
- **戰鬥派生屬性**：由六維推導而出的實際戰鬥數值，如：HP (體質x10)、攻擊力 (力量x2)、命中/閃避 (敏捷與幸運的公式)。
- **個別特訓系統**：在**「⚔️ 訓練所 (Camp)」**中，可針對單一冒險者花費金幣 (等級 × 50) 進行一對一指導，直接獲取升級經驗值。
- **招募系統**：具備「三選一」的類抽卡介面，確保玩家可以針對當前隊伍的短板進行人才篩選。

## ⚔️ 4. 裝備與戰利品系統
強調隨機性與模組化的打寶體驗：
- **戰利品掉落 (Loot)**：在野外將冒險者派遣「討伐魔物」，任務結算時有機率掉落裝備。
- **隨機詞綴引擎 (EquipmentGenerator)**：
  - **模組化**：每種裝備底層都有一個 Template (如：鐵劍)。
  - **隨機屬性**：根據裝備等級，系統會隨機賦予不同的六維與戰鬥屬性加成點數，確保每一把掉落的武器都是獨一無二的 (擁有獨立 UUID)。
- **七大裝備槽位**：頭盔、護甲、手套、靴子、披風、主手、副手。
- **裝備穿戴門檻**：裝備無職業限制，但採用「六維門檻檢定」。例如某把大劍可能要求「基礎力量達 20」才能裝備。
- **裝備倉庫管理**：在**「🏠 自宅 (Base)」**中，您可以隨時點開裝備倉庫檢視所有家當，並在冒險者介面中自由更換武裝。

---

## 🔨 5. 裝備強化系統 (Enhancement)
為了讓玩家消化多餘的金幣並追求極限數值，在**「⚒️ 鍛造屋 (Forge)」**中實作了經典的 + 制強化：
- 每次強化成功，裝備會獲得 `+1` 字綴，並且戰鬥屬性會有感提升。
- **機率曲線**：
  - `+0 ~ +2`：100% 成功。
  - `+3 ~ +5`：80% 成功。
  - `+6 ~ +9`：40% 成功。
- **極限與懲罰**：最高可強化至 `+10`；而在 `+6` 以上的挑戰中，如果強化失敗，裝備將會面臨**「退 1 階」**的殘酷懲罰。

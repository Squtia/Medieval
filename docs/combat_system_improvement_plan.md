# 戰鬥系統改善建議計畫

更新日期：2026-07-21  
文件性質：建議與執行規劃，不代表已完成實作

## 一、改善目標

1. 讓 UI 顯示的戰力與實際戰鬥能力一致。
2. 修正可能造成不公平或不可預期結果的戰鬥公式。
3. 讓職業、屬性、陣型與敵人特性形成可理解的相剋關係。
4. 增加戰前配置及多波戰鬥的風險決策，不把遊戲改成複雜的即時操作。
5. 保留目前 Phaser 戰鬥演出與事件戰報架構，逐步擴充而非重寫。

## 二、優先級總覽

| 階段 | 目的 | 主要項目 | 建議優先級 |
|---|---|---|---|
| 第一階段 | 修正可信度 | 戰力估算、暴擊上限、行動順序、狀態堆疊 | P0 |
| 第二階段 | 建立角色差異 | 職業攻擊方式、MP 技能、前後排規則、陣型編輯 | P1 |

## 三、第一階段：戰鬥邏輯校正

### 3.1 重做派遣戰力估算

目前的總戰力直接加總角色屬性，但智力、精神與 MP 尚未產生對等的實戰效益，可能讓風險預覽失真。

建議：

- 戰力拆成「輸出、生存、命中／速度、技能效益」四部分。
- 技能系統完成前，降低尚未投入戰鬥公式的屬性權重。
- 風險顯示除低／中／高外，再列出主要原因，例如「命中不足」或「缺少前排」。
- 後續可用固定種子模擬多場戰鬥，校正預估戰力與實際勝率。

驗收條件：

- 高智力但沒有攻擊技能的角色，不會被誤判為高輸出角色。
- 同一隊伍、同一任務的風險評估結果保持一致。
- 風險提示能反映高防禦、高閃避等敵人特性。

### 3.2 修正行動順序

建議：

- 增加獨立的速度值，主要由敏捷與裝備決定。
- 每回合每名參戰者只擲一次先攻值，再依結果排序。
- 不在排序比較函式中重複抽亂數。

驗收條件：

- 固定亂數種子時，行動順序與戰鬥結果可以重現。
- 閃避不再同時過度影響防禦與先攻。

### 3.3 限制暴擊率

建議：

- 暴擊率設定合理下限與上限，初期可採 5%～35%。
- 長期將命中與暴擊拆成不同屬性。
- UI 顯示角色實際命中率與暴擊率，避免隱藏公式。

### 3.4 定義狀態效果堆疊規則

建議：

- 同種狀態預設刷新持續時間，不直接無限新增。
- 若允許堆疊，限制最多 2～3 層。
- 流血、中毒改由特定職業、技能或裝備觸發。
- 首領可具備狀態抗性，但不應完全免疫所有控制。

## 四、第二階段：職業與陣型深度

### 4.1 建立最小職業戰鬥套件

每個職業先提供：

- 一種普攻目標規則。
- 一個被動能力。
- 一個消耗 MP 的招牌技能。

建議範例：

| 定位 | 普攻／目標規則 | 被動或技能方向 |
|---|---|---|
| 戰士 | 優先攻擊前排 | 嘲諷、保護後排、減傷 |
| 弓手 | 可攻擊後排 | 集中射擊、對高閃避敵人加成 |
| 法師 | 智力轉換魔法攻擊 | 範圍攻擊、部分無視防禦 |
| 牧師 | 普攻較弱 | 精神轉換治療量、解除異常 |
| 盜賊 | 可突襲後排 | 暴擊、流血、處決低血量目標 |

原則：先讓每個職業有清楚用途，不急著一次加入大量技能。

### 4.2 開放出戰前陣型配置

建議：

- 玩家可在派遣畫面調整前排與後排。
- 前排有容量限制，避免全隊塞入同一排。
- 近戰通常只能攻擊前排；遠程、法術或突襲技能可攻擊後排。
- 派遣前提示「缺少前排」「後排暴露」等陣型風險。

## 五、第三階段：多波戰鬥與風險報酬

### 5.1 重新設計進度討伐

目前多波戰鬥風險提高，但獎勵沒有對應成長。建議採累進獎勵：

| 完成進度 | 建議累積報酬 |
|---|---:|
| 第一波 | 基礎獎勵 100% |
| 第二波 | 額外 40% |
| 第三波 | 額外 60%＋稀有掉落加成 |

### 5.2 加入波次間選擇

每波結束後可選擇：

- 繼續進攻。
- 使用有限補給。
- 調整陣型。
- 撤退並保留已取得的部分獎勵。

驗收條件：

- 繼續與撤退都可能是合理選擇。
- 多波模式的期望報酬明顯高於單波模式。
- 玩家能在決定前看到隊伍剩餘 HP、MP、異常狀態與下一波情報。

## 六、第四階段：地形與敵人機制

### 6.1 地形效果

初期只加入小幅、容易理解的修正：

- 平原：無修正，作為基準。
- 森林：遠程命中略降、閃避略升。
- 沙漠：MP 消耗或補給壓力增加。
- 雪山：速度降低。
- 火山：固定環境傷害或火焰效果提高。

單項地形修正建議控制在 5%～15%，避免壓過角色養成。

### 6.2 敵人情報與反制

- 戰前公開敵人的主要特性，不必顯示所有精確數值。
- 高防禦、高閃避、後排輸出、毒性攻擊應有不同組隊解法。
- 首領使用可辨識的階段機制，不只增加 HP 和攻擊力。

## 七、測試與平衡計畫

至少補齊下列自動化測試：

1. 暴擊率不會超過上下限。
2. 固定種子可重現行動順序及戰鬥結果。
3. 同種狀態符合刷新或堆疊上限。
4. 近戰、遠程與突襲的前後排目標規則正確。
5. 空隊伍、全滅、回合上限與狀態傷害致死均能正確結算。
6. 多波獎勵、撤退與部分戰利品計算正確。
7. UI 風險預測與大量模擬勝率的誤差維持在可接受範圍。

平衡資料建議記錄：

- 各難度的實際勝率。
- 平均戰鬥回合數。
- 各職業出場率與勝率。
- 每名角色造成傷害、承受傷害、治療與控制次數。
- 玩家在各波選擇繼續或撤退的比例。

## 八、建議實作順序

1. 戰力估算與風險預覽。
2. 行動順序與暴擊上限。
3. 狀態效果規則與測試。
4. 職業普攻規則及一個招牌技能。
5. 陣型編輯與前後排目標規則。
6. 多波累進獎勵與撤退選項。
7. 地形效果與進階敵人機制。
8. 依實際勝率資料持續調整數值。

## 九、本計畫暫不處理的項目

- 不改成即時操作戰鬥。
- 不重寫現有 Phaser 戰鬥演出架構。
- 不在第一輪一次製作大量技能與特效。
- 不以單純提高敵人 HP 作為主要難度手段。
- 不在缺少測試資料時進行大規模全域數值重平衡。

# 專案交接文件 (Handover Document)

這份文件用來記錄目前的開發進度、遇到的已知問題，以及接下來的待辦事項，方便下一次對話或是交接給其他人開發時能快速進入狀況。

## 📍 目前開發進度 (Current Status)
- **基礎架構已成型**：我們完成了一個可以用 TypeScript 與 Vite 直接執行的放置遊戲雛形。
- **視覺與場景框架完成**：實作了深色模式、玻璃擬物化風格的「世界地圖」與「國家場景」切換機制。
- **已實作系統**：
  1. 冒險者系統重構：擴展為七大裝備槽位，並加入 `CombatStats` 派生戰鬥屬性計算。
  2. 動態裝備檢定系統（無職業限制，但有六維穿戴門檻）。
  3. 冒險者派遣系統（包含戰力檢定與野外據點互動）。
  4. 離線收益結算引擎（自動補齊流失的回合）。
  5. 內政收稅與外交獻禮系統（謁見廳進貢與宴會）。
  6. 爵位自動晉升（由平民、騎士一路晉升至公爵）。
  7. 招募系統與戰力特訓功能，實作了「三選一」的隨機生成招募 UI。
  8. 數據模組化：建立 `DataStore.ts` 管理職業與裝備，建立 `NameGenerator.ts` 處理姓名生成。
  9. **倉庫與隨機裝備系統**：實作了帶有隨機屬性的戰利品掉落，並存入 `Territory.warehouse`，玩家可於 UI 自行替換裝備。
  10. **裝備強化系統**：實作了鍛造屋，允許花費資源進行裝備強化（帶有機率與失敗降階懲罰）。
  11. **獨立建築視圖重構**：將自宅、謁見廳、訓練所、鍛造屋抽離為獨立全螢幕視圖，實作進入/退出建築的三層式導覽。
  12. **動態沙盒地圖引擎**：實作了 `MapDynamicsSystem`，地圖節點與派系會隨時間自然擴張演化，玩家也能自由在荒野建立新據點。近期已大幅優化降速，加入節點功能特徵 (`OCCUPIABLE`, `SUBJUGATION`, `MONSTER_NEST`)、距離限制與玩家據點的絕對保護機制。
  13. **暗黑羊皮紙戰略風格與全新 30 節點大陸**：將科幻 UI 全面翻新為暗黑羊皮紙風格，並導入高寫實地形底圖，擴充大陸節點至 30 個並具備獨特 TerrainType，已將名稱與座標正確配對。
  14. **冒險者數據與裝備輕量化重構**：裝備槽位縮減為 3 格，冒險者等級上限設為 Lv.10，升級改為手動配點制，新增魅力 (charm) 與統帥 (command) 屬性（預留未來外交或兵團系統使用）。
  15. **退休轉任機制**：實作了冒險者退休功能，退休後脫離隊伍並根據其魅力永久微幅提升領地稅收。
  16. **戰鬥公式模組化**：派遣任務新增敵方特徵 (`EnemyFeature`)，戰鬥結算改用屬性對抗邏輯（例如高防禦需高攻擊破解，高閃避需高命中破解）。

## 🔍 已知問題或待優化 (Known Issues)
- 當前所有的國家設定都是寫死在 `main.ts` 中的陣列，未來應移出為獨立的 Config 檔或資料庫。
- 各獨立建築 (自宅、鍛造屋等) 的背景目前只有 CSS 漸層色佔位，尚無正式的美術背景圖。

## 📝 下一步計畫 (Next Steps)
接下來您可以考慮往以下方向繼續開發：
1. **美術升級 (Art & UI)**：將 `index.html` 中註記為 TODO 的獨立建築背景替換為實際的美術圖 (如 `bg-forge.png`)。
2. **冒險者特訓擴充 (Advanced Training)**：在訓練所中，將「全面升級」擴充為針對特定屬性 (如力量、敏捷) 的特別重訓方案，並給予不同的收費標準。
3. **裝備改造系統 (Modification/Reforge)**：使用特殊道具改變裝備特性，例如重置隨機屬性池。
4. **建築系統 (Buildings)**：實作如何花費金幣解鎖 `Territory.unlockedBuildings` 並賦予全域加成（例如：酒館增加招募機率、訓練所增加戰力）。
5. **隨機事件系統 (Random Events)**：在 `resolveTurn()` 內政結算中加入隨機天災或豐收的事件，增加政治經營的變數。
